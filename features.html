Key Features of Ruby

Object-Oriented Programming

In Ruby, everything is an object, even primitive types. This consistent design choice allows for intuitive programming and object-oriented capabilities. 

          
            # Numbers and strings are objects
            puts 5.class       # Output: Integer
            puts "Hello".class # Output: String

            # Methods can be called on them
            puts 5.even?       # Output: false
            puts "hello".upcase # Output: HELLO

Simple and Readable Syntax

Ruby's syntax is designed to be easy to read and write. It is designed to be more natural and intuitive which reduces the learning curve and allows the developer to focus on solving problems. 

          
            # Simple and natural
            name = "Ruby"
            puts "Hello, #{name}!"  # Output: Hello, Ruby!

            # Control structures
            if name == "Ruby"
              puts "You're learning #{name}!" # Output: You're learning Ruby!
            end

Dynamically and Strongly Typed

Ruby is dynamically typed, which means variable types don't need to be declared. Ruby's "duck typing" references the phrase "if it walks like a duck and quacks like a duck, then it must be a duck". In simple terms, this means there is a focus on what an object can do, rather than its class, which provides more flexibility. 

          
            def quack(animal)
              if animal.respond_to?(:quack)
                animal.quack
              end
            end

            class Duck
              def quack
                puts "Quack!"
              end
            end

            quack(Duck.new) # Output: Quack!

Built-In Iterators and Blocks

Ruby provides tools like iterators, blocks, and enumerable methods, making it easy to work with collections. Tasks like sorting, filtering, and mapping are made easy with Ruby's syntax. 

          
            # Iterate over an array
            [1, 2, 3].each do |num|
              puts num * 2
            end
            # Output:
            # 2
            # 4
            # 6

            # Map to transform an array
            squared = [1, 2, 3].map { |num| num ** 2 }
            puts squared.inspect  # Output: [1, 4, 9]

Metaprogramming

Ruby allows you to write code that modifies itself, which enables the developer to use advanced techniques such as dynamic method creation. This feature is the reason why Ruby on Rails exists in its current form. 

          
            class Person
              attr_accessor :name

              def initialize(name)
                @name = name
              end

              def self.add_greeting(greeting)
                define_method(greeting) do
                  puts "#{greeting.capitalize}, #{@name}!"
                end
              end
            end

            Person.add_greeting(:hello)
            person = Person.new("Bob")
            person.hello  # Output: Hello, Bob!

Extensive Standard Library

Ruby comes with an extensive set of libraries for handling common tasks like file I/O, string manipulation, regular expressions, and more, so you can easily get started without worrying about building your own tools.

Open Classes

Ruby allows classes to be modified or extended at runtime. This functionality can be useful and dangerous at the same time so it is best to proceed with caution when using it or to avoid it all together. If used, it is important be certain that method names do not conflict with pre-existing, or future names. Overuse of open classes encourages poor design so it is preferred to follow best practices in order to avoid using them.

Exception Handling

Ruby handles exceptions using an easy to understand error handling system that uses 'begin', 'rescue', and 'ensure' blocks. 

          
            begin
              10 / 0
            rescue ZeroDivisionError => e
              puts "Error: #{e.message}"  # Output: Error: divided by 0
            ensure
              puts "This always runs."
            end

Ecosystem

Ruby still has an active community with a large collection of Gems (Ruby Libraries) that can help you add various functions to your programs with ease.

Frameworks

Ruby's most popular framework, Ruby on Rails, is designed to simplify the development of web applications. The philosophy behind Rails is Don't Repeat Yourself and Convention Over Configuration. In addition to this, Rails allows for rapid prototyping thanks to its application generator. Rails was once the go-to framework for web development but has fallen in popularity due to the success of other frameworks and technologies, however, it is still widely used and many apps are built on Rails. Below is a simple example of how to generate a new Ruby on Rails site. 

          
            # 1. Create a new Rails application
            rails new my_rails_app
            
            # Navigate into the application directory
            cd my_rails_app

            # 2. Run the Rails server to check the app (default port: 3000)
            rails server
            
            # Access the app in your browser: http://localhost:3000
            
            # Stop the server (Ctrl+C) before generating resources

            # 3. Generate a scaffold for an Article resource
            rails generate scaffold Article title:string body:text

            # This will create:
            # - Model (app/models/article.rb)
            # - Controller (app/controllers/articles_controller.rb)
            # - Views (app/views/articles/)
            # - Database migration (db/migrate/TIMESTAMP_create_articles.rb)
            # - Routes (config/routes.rb)

            # 4. Apply database migrations to update the schema
            rails db:migrate

            # 5. Start the Rails server again
            rails server

            # Visit the Articles resource at http://localhost:3000/articles
